# Inspirations

Works, ideas, and conventions that informed the look and approach of
this project. These are *not* sources that were directly used (those
live in [[../credits/_index]]); they're the cultural / aesthetic
context.

## Generative art lineage

- **The Vogel sunflower pattern** in particular is one of the
  most-replicated generative-art demos of the last 40 years —
  it's been re-implemented in Processing, p5.js, ShaderToy, Houdini,
  TouchDesigner, and probably every creative-coding language ever
  released. Implementing it once is a rite of passage. See
  [[../credits/vogel_phyllotaxis]] for the original source.
- **Procedural trees / L-systems** as a graphics tradition trace
  back to Prusinkiewicz & Lindenmayer's *The Algorithmic Beauty of
  Plants* (1990, free online: http://algorithmicbotany.org/papers/abop/abop.pdf).
  Every "draw a tree" tutorial in graphics since then is downstream
  of that book.

## Wallpaper conventions

- **macOS dynamic wallpapers** (Mojave's "desert," Catalina's
  "island") set the aesthetic floor: deep, low-saturation
  backgrounds with focal elements that don't fight UI chrome. The
  palettes here are tuned the same way.
- **Apple's "Aerial" screensaver** (the drone footage one) is the
  reference point for "calm enough that you forget it's moving." If
  you notice a wallpaper, it's failed.

## Color theory pointers (not sources, just useful reading)

- Josef Albers, *Interaction of Color* (1963). The reference on how
  colors behave next to each other. The palettes in this project use
  analogous tones + one sparing accent, which is straight Albers.
- Inigo Quilez's blog (https://iquilezles.org/) has dozens of writeups
  on color, easing, and procedural patterns that anyone doing this
  kind of work should bookmark.

## Manim cookbook references (not sources, but pedagogically useful)

- The Manim Community docs (https://docs.manim.community/) for the
  API surface used.
- 3Blue1Brown YouTube videos for examples of what's possible — though
  the educational-video use case is the opposite of the wallpaper use
  case (those want demand attention; we want to not).

## What this project tried *not* to be inspired by

- **Crypto / Web3 aesthetic** — neon gradients, glassmorphism,
  rotating glyphs. Sells well in 2026 but ages worst.
- **"Aurora" wallpapers** with vibrant gradient flows. Beautiful but
  busy; saturated colors at full opacity fight UI chrome.
- **Anything with text or branding**.
