The rules · what does what

  • Tabs stay top-level only · field, decisions, journal, credits. Sub-folders never become tabs.
  • Stepper walks every note at any depth in that tab's folder, ordered by sort_order then alphabetical path. With 3 files at depth 1 and 2 files in field/decisions/, that's 5 total inside the field tab.
  • Position indicator grows a path tail when the current note is nested · "3 of 5 · field" at depth 1, "3 of 5 · field › decisions" at depth 2.
  • Note breadcrumb chip · only appears above the note title when the note is nested below the tab folder. At depth 1 it's redundant with the tab, so it's hidden. Keeps the depth-1 case (most common) uncluttered.
  • End-of-folder · left arrow on first note is grayed out (disabled), right arrow on last note is grayed out. No wrap-around — matches the linear-reading mental model.
  • Deeper nesting · works the same · "field › decisions › 2026 › Q2" if you ever go that deep. We can truncate the middle with "…" if the chain exceeds 3 segments to keep the chip readable.

Code change implied

Current ProjectNotes.astro at line 27 flattens nested paths into the top group name and discards the rest. The new logic preserves the full path: field/decisions/ADR-001 → { tab: "field", subPath: ["decisions"], leaf: "ADR-001" }. The flattened list per tab is built from that, sorted, and handed to the stepper.
