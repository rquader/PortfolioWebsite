What changed

  • One notes button · the dark terracotta filled button is the popup trigger. It shows total file count, not per-folder counts (the popup handles folder nav).
  • Modular link row · only the buttons matching available frontmatter render. AdhkarCounter has github + download + demo; Arabic Map has github + live site. Easy to add new link types (app store, paper, etc.) by extending the schema.
  • Story / Info toggle · segmented control in the page header. Story mode and Info mode each read from their own frontmatter field on the project (e.g. info_md vs story_md) — same page shell, different text content.

Two design choices to confirm

  1. Notes button label · "open notes / 7 files" with the ◰ glyph reads like a folder. Alternatives: "field notes", "the working notes", "behind the work", or a verb like "look inside". Which voice fits?
  2. Story-mode wiring · Two ways to source the story text:
    (a) each project has a story_md field (per-project segments stored with the project — easier to keep colocated), or
    (b) one master story.md file with anchors per project (one continuous document — easier to read as one narrative when writing).
    I'd recommend (a) for now since it keeps the data model symmetric, but (b) better matches "a story that goes thru all the apps." Preference?

Once we lock these two, we move to the popup design — the fun part.
